QA 对照

交易确认页：改动前后对照

用于对比旧版确认页与新版 Security Checks / Transaction Preview 布局的静态场景矩阵。

Security Checks 会收拢重复的安全提示内容。地址 badge 和底部风险确认仍保留在原位置。只有 warning / critical 地址 tag 会汇总进卡片。

Security Checks 展示规则

什么时候展示

有安全项时展示。没有安全项时，等站点与本次操作分析都有结果后才展示 No issues。

默认展开

有 critical、warning，或非站点类 unverified / unknown 项时默认展开，例如 raw message data。

默认收起

只有站点 Unverified 时默认收起。用户仍能在卡片右侧看到 Unverified badge。

保留原位置

地址 badge 留在地址行。底部风险确认留在 footer。Transaction Preview 不合并进 Security Checks。

右侧摘要计数

critical 显示为 Risk，warning 显示为 Warning。两者同时存在时分开计数，不把 warning 算进 Risk。

异步状态

站点或本次操作分析未返回时不展示卡片，也不提前显示 No issues。 后续异步出现 warning / critical 时自动展开。
